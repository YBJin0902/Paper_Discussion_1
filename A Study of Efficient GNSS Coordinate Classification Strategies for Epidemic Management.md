# A Study of Efficient GNSS Coordinate Classification Strategies for Epidemic Management

研究背景（Introduction）

在疫情期間，政府與防疫單位需要追蹤民眾的移動軌跡，以辨識群聚地點與潛在感染風險。然而，大量 GNSS（全球導航衛星系統）資料帶來了龐大的資料分類與處理挑戰。
因此，本研究以**GNSS 座標分類演算法**為核心，探討如何在**大規模定位資料**中有效判斷目標位置，進而支援疫情熱區監控與人流管理。

相關研究（Related Work）

### PIP Point-in-Polygon

此方法用於判斷一個座標點是否位於多邊形區域內，常見於地理資訊系統（GIS）。
PIP 演算法主要有兩種方式：

* **射線法（Ray Casting Method）**：計算從該點向外射出射線，與多邊形邊界相交的次數。若相交次數為奇數，代表點在內部；偶數則在外部。
* **繞行數法（Winding Number Method）**：根據邊界繞過該點的方向次數判定。

### KNN

KNN 是一種常見的機器學習分類方法。
原理：

* 對於一個測試點，尋找訓練集中**距離最近的 k 個點**。
* 根據鄰居的多數類別投票結果，判定測試點的分類。

**優點**：簡單、直觀、能應用於地理資料分類。

**缺點**：運算量大，尤其在資料點龐大時效能下降。

## 系統模型

整體系統分為兩階段：

### 定位階段

1. 行動裝置（目標）回傳 GNSS 座標點至伺服器。
2. 伺服器判定該點是否位於指定區域（透過 PIP）。
3. 將分類後的點儲存為訓練資料集。

### 分類階段

1. 伺服器接收即時回傳的 GNSS 資料。
2. 伺服器從已分類的資料集中找出候選區域。
3. 使用 KNN 分類器將新點歸入對應區域，達到快速分類與追蹤效果。

## 方法

研究者設計了三個核心演算法：

### Algorithm 1：**SegSegInt**

計算兩條線段是否相交（用於判斷射線與邊界是否交叉）。

### Algorithm 2：**PIP_EP**

輸入一個點與多邊形，使用 Algorithm 1 的結果來判定點是否在多邊形內。

### Algorithm 3：**PtPos**

將目標點與多個多邊形集合比較，輸出該點所在的多邊形索引。

這三個演算法組合實現了完整的 PIP 檢測流程。

## 實驗與應用場景

研究團隊以台灣的地圖作為案例：

* (a) 熱點分佈（約 6024 筆 GNSS 點）
* (b) 高速公路與都市區域的座標分佈

透過 PIP + KNN 的整合，可快速分類與定位疫情期間的熱點範圍，有助於：

* 疫情足跡追蹤
* 區域風險分級
* 即時警示系統
